传统GMM-HMM

  1. 传统的GMM-HMM模型里，首先需要对帧和标签进行对齐。也就是需要知道哪几帧是什么音素.
    传统模型里是通过GMM(混合高斯模型)的EM算法来实现的.
    是一个非常耗费时间，需要反复迭代的过程
  2. EM算法第一步先求出要估计参数的粗略值，第二步使用第一步的值最大化似然函数

React Study

第一天

  1. React的特点

    • 组件化模式，声明式编码
    • 使用虚拟DOM＋Diffing算法，减少与真实DOM的交互

  2. React 文件

    • babel.min.js: JSX转JS
    • react.development.js: 核心库
    • react-dom.development.js: 操作DOM

  3. JSX

    • 标签中混入JS表达式时需要用{}
    • 样式的类名指定不要用class，要用className
    • 内联样式要用style= {对象}的形式
    • 只有一个根标签
    • 标签必须闭合
    • 组件名称的首字母要大写，如果首字母会小写会自动去匹配html的原生元素

  4. state

    • state不可以直接更改，要借助setState来更改
    • 自定义方法要用赋值语句的形式： changeWeather = () => {}

  5. props

    • 批量传递props: <Person {…p}> //p是一个对象
    • 对props进行限制 Person.propTypes = 对象,
    • 对props设置默认值 Person.defaultProps=对象
    • props 是只读的
    • 函数式组件使用props：用参数来进行传递

  6. ref

    • 字符串ref(不推荐，之后会被移除)，通过this.refs来获得
    • 回调ref
      • 为了清空旧的ref，在更新的时候会调用ref两次，第一次传入null
    • crate ref: 相当于一个容器，专人专用，只能存一个

  7. 受控组件和非受控组件

    • 非受控组件：数据随用随取，一般用ref
    • 受控组件：提前存储数据，一般用state

  8. 高阶函数

    • 函数接收的参数是一个函数
    • 函数的返回值还是一个函数(通过函数调用继续返回函数，实现多次接收参数
      最后统一处理的方式叫做函数的柯里化)

  9. 脚手架
